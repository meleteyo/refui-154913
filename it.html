FAQ · 자주 묻는 질문
1개월 무상 유지보수 후엔 어떻게 되나요? ▾
무상 유지보수 기간(1개월) 종료 후에는 유상 유지보수(월정액 또는 이슈별 견적) 계약으로 전환 가능합니다. 인수인계 문서·배포 스크립트·GitHub 이전을 완료하므로, 별도 파트너 없이 자체 운영도 가능합니다.
코드 소유권은 누구에게 있나요? ▾
최종 납품 이후 모든 소스코드·산출물의 소유권은 발주자에게 이전됩니다. 위시켓 표준 계약 기준으로 진행하며, 별도 라이선스 제한 없습니다. GitHub Private Repo 이전으로 완료합니다.
다른 개발자도 코드를 받을 수 있나요? ▾
네. 헥사고날 + 모듈 분리 구조로 설계하며, Spring Boot 표준 프로젝트 구조를 사용합니다. 인수인계 문서(README + API 명세 + ERD + 배포 가이드)를 포함하므로 다른 개발자도 인수 가능합니다.
인수인계 품질은 어떻게 보장되나요? ▾
M8(W22)에 산출물 패키지를 제공합니다: 소스코드 + OpenAPI 명세 + ERD + 배포 스크립트 + 운영 가이드 + 1개월 무상 유지보수. 인수인계 체크리스트를 미팅에서 합의합니다.
기술 종속성 우려는 없나요? ▾
Java 21 + Spring Boot 3.x + PostgreSQL + Next.js 모두 한국 시장에서 가장 많이 쓰는 표준 기술입니다. 특수 프레임워크·독점 SaaS 도구를 사용하지 않아 향후 다른 파트너도 유지보수 가능합니다.
Spring Boot 외 기술 옵션이 가능한가요? ▾
Python/FastAPI(분석 특화) 또는 Node/NestJS(풀스택 단일 언어) 모두 가능합니다. 다만 결제·구독 트랜잭션 안정성과 22년 Spring Boot 강점 활용을 고려할 때 Spring Boot 우위로 판단합니다. 미팅에서 구체 요구 확인 후 재산정 가능합니다.
마이크로서비스 분리가 가능한 구조인가요? ▾
네. Bounded Context 5개(identity / subscription / tenderview / tracking / admin)로 분리된 모놀리식으로 시작합니다. 헥사고날 + Port/Adapter 분리로 향후 MSA 전환 경로를 보존합니다. 트래픽·팀 규모에 따라 단계적 분리를 권장합니다.
